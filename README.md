# HASH-FUNCTION-COMPARISON-STUDY
Comparison for modular, polynomial rolling and FNV-1a hash functions
1. PROJECT TITLE

Hash Function Comparison Study Using Modular, Polynomial Rolling, and FNV-1a Hash Functions

2. PROJECT OVERVIEW

Hashing is a fundamental technique in computer science used to efficiently store, retrieve, and manage data. Different hash functions produce different distributions of keys across a hash table, which directly affects the number of collisions and overall performance of the data structure.

This project investigates and compares the performance of three commonly used hash functions:

Modular Hash Function
Polynomial Rolling Hash Function
FNV-1a Hash Function

The study measures how effectively each hash function distributes over 10,000 English words into a hash table of fixed size. The primary evaluation metric is the number of collisions generated by each hash function. Execution time is also measured to analyze computational efficiency.

The project demonstrates practical applications of hashing and provides empirical evidence of how hash function design influences hash table performance.

3. PROBLEM STATEMENT

Hash tables rely heavily on the quality of the hash function used. Poor hash functions may cause many keys to map to the same table index, resulting in excessive collisions and degraded performance.

The challenge is to determine which hash function provides the best key distribution and the fewest collisions when storing a large set of English words in a hash table of fixed size.

This project addresses this challenge by implementing and comparing three different hashing techniques under identical conditions.

4. OBJECTIVES
Main Objective

To compare the effectiveness of Modular, Polynomial Rolling, and FNV-1a hash functions using collision analysis.

Specific Objectives
Implement three different hash functions from scratch.

8. ALGORITHMS USED
Modular Hash Function

The ASCII values of all characters in a word are summed and reduced using modulo arithmetic.

Purpose:

Simplicity.
Baseline comparison.

Advantages:

Easy to implement.
Fast execution.

Disadvantages:

Poor distribution.
High collision rate.
Polynomial Rolling Hash Function

Each character contributes to the final hash value based on powers of a constant multiplier.

Purpose:

Improved key distribution.

Advantages:

Lower collision rate.
Commonly used for strings.

Disadvantages:

Slightly slower than modular hashing.
FNV-1a Hash Function

Uses XOR and multiplication operations to generate hash values.

Purpose:

High-quality distribution.

Advantages:

Excellent dispersion.
Low collision rate.
Industry-standard algorithm.

Disadvantages:

More computational steps.

10. CONCLUSION.

This project provides a practical evaluation of three widely used hash functions. By comparing collision counts and execution times using a common dataset, it demonstrates the importance of selecting an appropriate hash function for efficient data storage and retrieval. The results are expected to show that FNV-1a offers the best distribution and lowest collision rate, while the modular hash function provides the weakest performance due to its simplistic design. This study reinforces the role of hashing in modern computing systems and highlights how algorithm choice affects overall system efficiency.
